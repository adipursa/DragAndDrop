# 디렉토리 관리 시스템 개선 내용

## 문제점
- 기존 코드는 디렉토리 ID 관리 맵과 화면 이벤트 관리 맵이 분리되어 중복된 디렉토리 구조를 관리하고 있었음
- 이로 인해 여러 파일에 같은 기능이 분산되어 있고 데이터 일관성 유지가 어려웠음

## 해결 방법
1. 모듈화된 여러 파일들을 하나의 통합된 파일로 병합
   - directoryCore.js, directoryEvents.js, directoryUI.js 등의 파일들을 directoryManager.js로 통합
   - 모든 관련 기능을 하나의 모듈에서 관리하도록 개선

2. 맵 구조 개선
   - directoryIdMap: 디렉토리 ID로 디렉토리 객체 조회
   - directoryPathMap: 경로 문자열로 디렉토리 ID 조회
   - 단일 책임 원칙에 따라 각 맵이 명확한 역할을 수행하도록 구성

3. 기능별 코드 구조화
   - 데이터 관리 함수
   - 이벤트 관리 함수
   - UI 조작 함수
   - 디렉토리 추가/삭제 관련 함수
   - 드래그 앤 드롭 기능

4. HTML 파일 업데이트
   - 모듈화된 여러 JS 파일 대신 통합된 directoryManager.js 파일을 사용하도록 변경

## 장점
1. 데이터 일관성 유지
   - 단일 데이터 소스를 통해 디렉토리 정보 관리
   - 모든 맵이 한 모듈 내에서 동기화되어 불일치 방지

2. 코드 중복 제거
   - 여러 파일에 분산된 유사 기능들을 통합하여 코드 중복 감소
   - 유지보수성 향상

3. 성능 개선
   - 파일 로드 감소로 초기 로딩 속도 향상
   - 여러 맵 간의 데이터 동기화 오버헤드 감소

4. 모듈화된 설계 유지
   - 하나의 파일 내에서도 기능별로 명확한 구조 유지
   - IIFE 패턴을 사용하여 전역 네임스페이스 오염 방지
